1 · O que o LAB8 adiciona

O LAB8 parte do processador dos labs anteriores e adiciona o uso real de memória de dados. O programa não calcula apenas dentro dos registradores: ele preenche a RAM, marca valores compostos com zero e depois lê a tabela pronta.

RAM

Memória de dados

Guarda a tabela do crivo. Endereço e dado são escolhidos por registradores do banco.

DJNZ

Loop regressivo

Decrementa o contador e salta se o resultado ainda não é zero.

exception_o

Validação de ROM

Sinal de exceção no top level, usado para indicar execução fora do espaço válido.

Flags

Controle por resultado

A ULA gera Zero, BLE e BCC; a UC usa a Zero flag registrada para decidir o DJNZ.

Resultado observado na simulação

O testbench imprime a sequência lida de R1 durante a fase final: 2, 3, 0, 5, 0, 7, 0, 0, 0, 11, 0, 13, ... 29, 0, 31, 0. Os zeros são os compostos que o programa marcou na RAM.

3 · Formato das instruções

Todas as instruções têm 15 bits. Os campos mudam de significado conforme o opcode: registradores para operações de ULA e memória, constante de 5 bits para LD/SUBI, ou endereço para branches e DJNZ.

[10:8] Rddestino / endereco
[7:5] Rs1fonte 1 / dado
[4:2] Rs2fonte 2
[4:0] cte5constante / parte do endereco

0010ADD Rd, Rs1, Rs2Soma duas fontes e grava em Rd.

4 · Programa do crivo

O programa usa registradores fixos para simplificar a validação. R0 vale 1, R7 vale 0, R6 aponta para a RAM, R5 conta loops com DJNZ, e R1 recebe o valor lido no final.

SetupR0=1, R7=0, R6=2, R5=31.
PreencherRAM[2..32] recebe o próprio índice.
MarcarMúltiplos de 2, 3 e 5 viram zero.
LerLW copia RAM[R6] para R1.
ExibirADD R1,R1,R7 coloca R1 em ula_out.
; trecho final da ROM - fase de leitura
48: LD   R6, 2       ; ponteiro inicial da tabela
49: LD   R5, 31      ; quantidade de endereços lidos: 2..32
50: LW   R1, R6      ; R1 = RAM[R6]
51: ADD  R1, R1, R7  ; R7=0, então ula_out mostra R1
52: ADD  R6, R6, R0  ; próximo endereço
53: DJNZ R5, 50      ; repete enquanto R5 != 0
54: JMP  -1          ; fim: loop infinito

5 · Por que q_r1 mostra a sequência certa

Na fase final, o valor que interessa sai da RAM e entra em R1. O caminho é: RAM[R6] -> ram_out_s -> data_wr_s -> banco_regs.R1. A UC ativa wr_en_reg_o no estado de execute da instrução LW, então o valor aparece em q_r1 depois da borda de clock.

Por que fica zero por tanto tempo?

Antes da leitura final, R1 quase não é usado; ele permanece em zero durante todo o preenchimento e a marcação da RAM. Depois, na fase de leitura, os compostos consecutivos também aparecem como zero porque a RAM foi zerada nesses endereços.

Intervalo entre valores

A máquina de estados roda 00 -> 01 -> 10, então cada instrução custa 3 ciclos. Uma iteração do loop de leitura executa LW, ADD, ADD e DJNZ. Quando o DJNZ salta, ainda há a bolha de flush para descartar a instrução buscada no caminho errado. Por isso um novo valor útil aparece a cada aproximadamente 15 ciclos. Com período de 100 ns, isso dá cerca de 1,5 us por posição lida.

Hexadecimal no GTKWave

O GTKWave costuma mostrar vetores em hexadecimal. Então pc_o = 31 na tela significa 0x31 = 49 em decimal. O loop que aparece como 31, 32, 33, 34, 35, 36 é, na verdade, o loop decimal 49, 50, 51, 52, 53, 54.

6 · Como o DJNZ funciona

O DJNZ foi implementado de forma direta: ele lê o registrador Rd, manda a ULA subtrair 1, grava o resultado de volta em Rd e decide o salto usando a Zero flag resultante. 

-- ideia do caminho de dados
djnz_cte_s  <= "0000000000000001";
mux_b_s     <= djnz_cte_s when djnz_dec_s = '1' else ...;
ula_op_o   <= "01"; -- SUB
wr_en_reg  <= '1';  -- grava Rd decrementado
wr_en_flag <= '1';  -- guarda flag_z para o flush

Há um detalhe importante: no ciclo de execute, a UC usa ula_zero_i para decidir se deve escrever o novo PC. No ciclo seguinte, durante o fetch/flush, ela usa flag_z_i, porque a flag já foi registrada e o banco de registradores já mudou. Isso evita que o flush seja decidido com o valor errado.

7 · Memória: SW e LW

A RAM tem 128 posições de 16 bits. No SW, o endereço vem de um registrador e o dado vem de outro. No LW, o endereço vem de um registrador e o valor lido volta para o banco de registradores.

InstruçãoEndereço usadoDadoUso no crivo
SW R6, R6 R6 R6 Preenche RAM[n] = n.
SW R6, R7 R6 R7 = 0 Zera um número composto.
LW R1, R6 R6 RAM[R6] Lê a tabela pronta para exibir em R1.

10 · Checklist de instruções

Este é o programa assembly curto que pode ser usado como referência para conferir a execução de todas as instruções implementadas ao longo do semestre. 

; checklist manual da ISA do grupo
NOP
LD   R0, 1        ; constante
LD   R1, 5
LD   R2, 3
ADD  R3, R1, R2   ; R3 = 8
SUBI R4, R3, 1    ; R4 = 7, atualiza flags
MOV  R5, R4       ; R5 = 7
SW   R5, R3       ; RAM[7] = 8
LW   R6, R5       ; R6 = RAM[7] = 8
SUBI R6, R6, 8    ; R6 = 0, Z=1
BLE  label_ble    ; branch condicional
BCC  label_bcc    ; branch condicional
LD   R7, 3
loop:
DJNZ R7, loop     ; decrementa ate zero
JMP  -1          ; fim
